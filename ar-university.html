AR University


What I did:


Developed the UI/UX (most recent draft) for a learning module to built on iOS devices w/ compatibility for Android. App was designed to be used as a learning tool where users are given a list of modules from which they can pick from, where within each module, users can select a variety of different lessons (experiences) - Augmented Reality, Video, Image, Document - where they can consume and learn from. Inside the experience, users can then find other type experiences related to the current lesson. Available modules depends on the user's subscription which has yet to be implemented.

Log in + Home Page


The log in is a standard login using the colour navy blue for a serious tone that resembles the intensity and serious tone of university. Home page is a page depicting a list of available modules based on the user's subscriptions. This was designed such that users can browse through what they have and may find interesting. A module page simply has a list of different lessons to choose from, with the types being AR, Video, Images, and Documents, which can filtered using the 'Show' button.





Experiences


Experience types available on the app to view through a module (brand) can be viewed as such:


Augmented Reality
Video
Image
Document


The Augmented Reality experience uses an AR onboarding located on the top when the camera is enabled with UIs to disable sound, receive additional info on how to use (AR), a back button to return to the previous module/lesson and an options to reveal information specifically about the experience.


The Video experience is most similar to YouTube's UI with the only exception of a related categories located at the bottom designed in the matter of Google's Search engine with filter for seach, images, etc. Upon clicking the related content the screen transitions back to the search results screen and traverses for content similar to the current experience/result currently on. Double tapping/hovering over the video will present additional video UI for based on Apple's default video player. This is designed to give users the ability to quickly toggle between the type of experience and access related content. Title of a module is position on the top of the device with the lesson name underneath it.


The Image experience is designed in the exact manner as the video experience but in image format. Double tapping the image will fullscreen the image (with UI to exit once fullscreen)

The Document experience is simply portrayed in PDF form with headings for the module and lesson name on the top (that truncates upon scrolling down). Images are also contained within context of the document can also be double tapped to view fullscreen. Once the user reaches the end of the document the related contents will reveal itself like a footer and guide users to enjoy further experiences.
