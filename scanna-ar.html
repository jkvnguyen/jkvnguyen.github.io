Scanna AR


Overview


Scanna AR is an Augmented Reality app that simulates a real life budtender to provide users with the ability to ask questions and details about Cannabis products. The app is built for iOS tablets and with Unity’s ARFoundation framework (built in Unity). App uses an image tracker which “scans” a product and identifies which product the budtender will assist the user with. The tracking is followed up by a markerless AR where a plane (in this case horizontal) is tracked by a camera and the user is prompted to tap the projected plane to summon the experience.

Note: App is only a proof of concept and not a fully built app.




What I did:


  • Implemented a markerless Augmented Reality experience with horizontal plane detection that activates an assistance bot explaining cannabis products
  • Designed a 3D environment for individual Cannabis products explaining product information in an intuitive and easy way
  • Added scanning features where scanning specific products enables a markerless AR experience that activates a hologram budtender that educates users on that particular cannabis product and about cannabis in general
  • Shot video demos of Scanna and cut together clips and audio to make a pitch video that was used during pitches by sales


Prefabs


The app includes three prefabs which are the three products that are currently on the app.

  1. Yin Yang
  2. Penelope Oil
  3. Tropical Breeze

Each prefab has a corresponding UI that when the user taps a ‘Learn More’ button, a modal box shows up with extra info and purchase potential. Each prefab contains content for a product including its THC, CBD, Name, Brand as 3D objects, and options for back, forward, and a ‘Learn More’.

Unity Scene


The main scene is Demo 2 which does all the AR tracking, markerless AR, and UI elements.

  1. AR Camera
    • ARFoundation pre-built, responsible for rendering the AR scene
  2. AR Session Origin
    • Contains PlaneManager, TapToPlace script, Raycast Manager, virtual button (interaction), image tracker, image info tracker
  3. Canvas: contains all the UI elements
  4. Lean Touch: enables gesture inputs
  5. Placement Indicator: prefab for when there is plane detection

Workflow


The main user workflow with the app is as follows:

  1. User points device at product barcode
    • Device detects the image from a reference library and chooses which sequence to play. Currently the products that work for scanning are Liiv Yin & Yang, SynrG Tropical Breeze, and Tweed Penelope Oil. (Products barcode may depend on how they look)
  2. User taps to start the AR experience
    • Once the device detects the image the user is prompted to point their device to a flat horizontal surface and a marker will appear
    • Tap on the marker to start the experience
  3. User taps to start the AR experience
    • The AR budtender explains features of the Cannabis product such as CBD, THC, etc.
    • Users can skip through various parts of the by using a back and forward button
    • A learn more button can be tapped to prompt more information about the product and the ability to visit the provider websites where they can buy.
  4. Static UIs
    • Scan: to reset the scanned product so that the user can go to another experience
    • Help: are answered by the hologram budtender (not implemented)
    • Discover: a feature from the Android app that allows the user to browse through more products (not implemented)
  5. Gesture Touches
    • Transform: transform moves the experience such that obstructive space can be moved so that the experience can be seen more clearly. This uses a 3 figure pan to move.
    • Scale: scaling helps with being able to see specs of the physical product more closely and allows for obstructive space to be more easily viewable. This uses a pinch in and pinch out gesture.
    • Rotate: rotate simply rotates the cannabis product such that the user can view specs about the potential physical properties of the product. Rotate uses a simply drag (only horizontally)

Scripts + Programmer Notes:


Image Tracking / Scanning: this is done using Unity’s ARFoundation in Image Tracking where images are simply uploaded to a reference library. Tracking to instantiate a prefab is achieved using Unity’s sample script TrackedImageInfoManager where depending on which image is tracked the corresponding prefab will be instantiated.

Tap To Place script: uses raycasting to shoot toward the center of the AR camera to track anything trackable. Interaction is limited to everything within the trackable polygon and this script is complemented by a DisableVerticalPlace script (part of Unity’s sample code) to ensure only horizontal tracking is achieved.

Video Controller script: a controller that sets video clips of budtender questions to the next or previous clip

Destroy a prefab: The scan UI simply reloads the Scene which should destroys prefabs created from instantiated

Efficiency Notes: UI could be instantiated and destroyed to improve efficiency as opposed to being set and turned active
